## 📊 外部需求看板使用规范

### 看板访问权限
- **允许**: 创建新需求、编辑自己创建的需求、添加评论
- **禁止**: 修改状态(状态由张良审核后修改)、删除需求、直接指派任务

### 数据库字段规范

| 字段 | 说明 | 09:00登记 | 21:00更新 |
|------|------|-----------|-----------|
| **需求标题** | 简明扼要的需求描述 | ✓ 必填 | 可补充 |
| **来源** | 具体来源渠道 | ✓ 必填 | - |
| **来源详情** | 客户名称/联系人/联系方式 | ✓ 必填 | 可补充 |
| **登记时间** | 自动记录 | 自动 | - |
| **登记人** | 郦食其 | 自动 | - |
| **详细描述** | 需求背景、具体诉求 | ✓ 必填 | ✓ 补充 |
| **期望交付时间** | 外部方期望的完成时间 | ✓ 必填 | ✓ 更新 |
| **紧急程度** | 外部方表述的紧急程度 | ✓ 必填 | - |
| **优先级建议** | 郦食其初步判断 | ✓ 选填 | - |
| **关联资料** | 相关文档/截图/链接 | 选填 | ✓ 补充 |
| **状态** | **待审核**/已审核/已拒绝/待补充 | **禁止修改** | **禁止修改** |
| **指派给** | **张良** | 自动 | - |
| **最新沟通记录** | 沟通进展 | - | ✓ 必填 |
| **更新时间** | 最后更新时间 | - | 自动 |

### 来源字段选项

```
来源:
├── 客户
│   ├── 客户A
│   ├── 客户B
│   └── 其他客户
├── 合作方
│   ├── 合作方A
│   └── 其他合作方
├── 邮件
├── 即时通讯(微信/钉钉/Slack)
├── 电话/会议
└── 其他
```

### 紧急程度定义

| 级别 | 定义 | 示例 |
|------|------|------|
| 极紧急 | 影响业务运行，需立即处理 | 系统故障、支付异常 |
| 紧急 | 有明确截止时间，延期影响业务 | 合同到期前功能、活动需求 |
| 一般 | 常规需求，可按排期处理 | 功能优化、新特性 |
| 低 | 建议类、长远规划类 | 未来版本建议、体验改进 |

### 优先级建议标准

| 建议 | 适用场景 |
|------|----------|
| P0建议 | 极紧急+重要客户/核心业务流程 |
| P1建议 | 紧急+常规客户/重要功能 |
| P2建议 | 一般需求/体验优化 |
| P3建议 | 低优先级/规划类需求 |

## ⚠️ 重要禁令

### ❌ 禁止行为

1. **禁止直接派发任务**
   - 不能绕过看板直接告诉开发人员做什么
   - 所有需求必须先登记到外部需求看板
   - 等待张良审核后才能进入开发流程

2. **禁止修改状态字段**
   - "状态"字段只能由张良修改
   - 登记时保持默认"待审核"
   - 即使与外部方确认过，也不能自行改为"已审核"

3. **禁止删除需求**
   - 登记错误也不删除，可标记作废
   - 保留所有记录便于追溯

4. **禁止修改他人登记的需求**
   - 只能编辑自己创建的需求条目
   - 如需补充他人登记的需求，添加评论

### ✅ 正确做法

| 场景 | 错误做法 | 正确做法 |
|------|----------|----------|
| 客户催需求 | 直接告诉开发加急 | 在看板补充"客户催办"备注 |
| 需求信息不全 | 随意填写后提交 | 标记紧急程度，在描述中注明"待补充信息" |
| 需求变更 | 直接修改原需求 | 添加评论说明变更，或新建条目关联 |
| 外部方询问进度 | 自行承诺时间 | 回复"已提交审核，将有专人回复" |
