# 张良(产品经理) Notion 看板使用规范

## 📋 职责概览
- 需求决策、优先级校准(10:00)
- 功能点验收(17:30)，失败打回Bug
- 向Owner汇报

---

## 🎯 每日定时任务

### 10:00 需求优先级校准
| 操作 | 看板位置 | 具体动作 |
|------|----------|----------|
| 读取外部需求 | 外部需求看板 | 查看最新登记的外部需求 |
| 审核需求 | 外部需求看板 | 更新需求状态(已审核/已拒绝/待补充) |
| 分配优先级 | 迭代看板 | 将审核通过的需求移至迭代看板并标记优先级(P0/P1/P2) |

### 17:30 功能点验收
| 操作 | 看板位置 | 具体动作 |
|------|----------|----------|
| 检查待验收项 | 迭代看板 | 筛选状态为"待验收"的任务 |
| 执行验收 | 迭代看板 | 验收通过则标记"已完成"，失败则转Bug看板 |
| 更新状态 | Bug看板 | 对失败项创建Bug记录，指派给相应开发人员 |

---

## 📊 看板使用规范

### 1. 外部需求看板(只读+审核)
**访问权限**: 读取全部，修改状态字段

| 字段 | 说明 | 张良操作 |
|------|------|----------|
| 需求标题 | 外部需求简述 | 确认理解无误 |
| 来源 | 客户/合作方/邮件/会议 | 核实来源真实性 |
| 登记时间 | 郦食其登记时间戳 | 检查时效性 |
| 详细描述 | 需求背景与具体内容 | 阅读并评估 |
| 期望交付时间 | 外部期望的时间 | 判断是否可行 |
| 状态 | **待审核** / 已审核 / 已拒绝 / 待补充 | **必填**，10:00前更新 |
| 优先级建议 | 郦食其初步判断 | 参考后重新定级 |
| 指派给 | 产品经理张良 | 自动分配 |

**状态流转**:
```
待审核 → 已审核(通过进入迭代看板)
     → 已拒绝(填写拒绝原因)
     → 待补充(注明需补充的信息)
```

### 2. 迭代看板(主要工作区)
**访问权限**: 完全控制

| 字段 | 说明 | 必填 |
|------|------|------|
| 任务名称 | 功能点描述 | ✓ |
| 迭代日期 | 格式: YYYY-MM-DD | ✓ |
| 优先级 | **P0** / P1 / P2 / P3 | ✓ |
| 状态 | 待开发 / 开发中 / 待测试 / **待验收** / 已完成 | ✓ |
| 负责人 | 开发人员 | ✓ |
| 预计工时 | 估点 | - |
| 实际工时 | 完成后填写 | - |
| 关联需求 | 链接到外部需求 | - |
| 验收标准 | 验收checklist | ✓ |
| 验收结果 | 通过 / 不通过(转Bug) | 17:30填写 |

**每日迭代视图设置**:
- 按日期分组
- 仅显示今日及未来3天
- 筛选状态≠已完成的任务

### 3. Bug看板(验收失败时写入)
**访问权限**: 创建Bug、更新状态

| 字段 | 说明 | 验收失败时填写 |
|------|------|----------------|
| Bug标题 | [验收失败] 功能名称-问题简述 | ✓ |
| 严重程度 | 阻塞 / 严重 / 一般 / 轻微 | ✓ |
| 状态 | 新建 / 修复中 / 待验证 / 已关闭 | 自动设为"新建" |
| 关联迭代任务 | 链接到迭代看板 | ✓ |
| 关联需求 | 链接到外部需求 | ✓ |
| 复现步骤 | 详细的复现方法 | ✓ |
| 期望结果 | 正确的行为描述 | ✓ |
| 实际结果 | 错误的行为描述 | ✓ |
| 截图/附件 | 问题截图 | 建议上传 |
| 报告人 | 张良 | 自动 |
| 指派给 | 原开发人员 | ✓ |
| 创建时间 | 验收时间 | 自动 |
| 修复截止时间 | 根据严重程度设定 | ✓ |

## 📈 向Owner汇报模板

每周五17:00汇总以下内容：

```markdown
## 本周工作汇报 (张良)

### 1. 外部需求处理
- 本周登记外部需求: X条
- 已审核通过: Y条 (进入迭代)
- 已拒绝: Z条
- 平均响应时间: X小时

### 2. 迭代完成情况
- 本周迭代任务: X个
- 按时完成: Y个
- 延期: Z个
- 验收通过率: X%

### 3. Bug情况
- 本周新建Bug: X个(验收失败产生)
- 已修复: Y个
- 待修复: Z个
- 严重Bug: X个(需Owner关注)

### 4. 下周计划
- 预计迭代任务: X个
- 重点需求: [列表]
- 风险点: [说明]

### 5. 需要决策的事项
- [事项1: 需要Owner拍板的内容]
```

## ⚡ 快捷操作

### Notion快捷方式
| 操作 | 快捷键 |
|------|--------|
| 新建页面 | Cmd/Ctrl + N |
| 搜索 | Cmd/Ctrl + P |
| 快速创建任务 | 在数据库中按 / 选择模板 |

### 推荐视图配置
1. **今日待办**: 筛选"待验收"状态的任务
2. **待审核需求**: 筛选"待审核"状态的外部需求
3. **本周迭代**: 按周分组显示迭代任务
4. **我创建的Bug**: 筛选报告人=张良的Bug
