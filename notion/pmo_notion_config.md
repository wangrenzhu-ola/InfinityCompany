## 二、各看板操作权限与字段规范

### 2.1 迭代看板（核心看板）

#### 视图配置
- **默认视图**：看板视图（按状态列分组）
- **辅助视图**：
  - 日历视图（按开始时间/结束时间）
  - 表格视图（用于批量编辑度量字段）
  - 今日迭代视图（筛选当日迭代）

#### 字段填写规范

| 字段名 | 类型 | 必填 | 填写时机 | 填写规范 |
|-------|------|-----|---------|---------|
| `任务名称` | Title | 是 | 创建时 | 简洁描述，格式：【模块】具体动作 |
| `状态` | Select | 是 | 全程 | 待协调/待开始/进行中/待验收/Bug/回流/已完成 |
| `负责人` | Person | 是 | 创建时 | 分配给具体角色（韩信/萧何/陈平等）|
| `Story 关联` | Relation | 是 | 创建时 | 关联父级 Story |
| `迭代日期` | Date | 是 | 创建时 | 格式：YYYY-MM-DD，每日新建迭代 |
| `token 开销(预估)` | Number | 是 | 任务开始前 | 预估 Token 消耗量（单位：千tokens） |
| `token 开销(实际)` | Number | 是 | 任务完成后 | 实际 Token 消耗量（单位：千tokens） |
| `开始时间` | DateTime | 是 | 任务开始时 | 实际开始时间戳 |
| `计划结束时间` | DateTime | 是 | 任务开始前 | 预估交付时间 |
| `实际结束时间` | DateTime | 是 | 验收通过后 | 实际验收通过时间 |
| `阻塞原因` | Text | 否 | 进入待协调时 | 描述阻塞的具体原因 |
| `优先级` | Select | 是 | 创建时 | P0（阻塞）/ P1（当日必做）/ P2（可延后） |
| `交付物链接` | URL | 否 | 完成后 | 关联 PR、文档或代码片段链接 |

#### WIP 限制监控
- 「进行中」列：每人最多 **2 个**任务，超限需预警
- 「待验收」列：最多 **5 个**任务，超过需优先处理验收

#### 定时关联操作

| 时间 | 仪式 | 看板操作 |
|-----|------|---------|
| 09:30 | 晨会 | 核对昨日看板，调整「进行中」任务状态 |
| 11:30 | 进度对齐 | 更新「进行中」任务进度，识别风险 |
| 17:00 | 日终评审 | 将验收通过任务移入「已完成」 |
| 17:20 | 看板归档 | 将「已完成」任务归档到当日迭代快照 |
| 19:00 | 晚间复盘 | 更新复盘记录，提炼改进项 |

### 2.2 Bug 看板

#### 状态定义（固定六种）

| 状态值 | 说明 | 流转条件 |
|-------|------|---------|
| **新增** | Bug 已登记，待确认 | 陈平发现 Bug 时创建 |
| **重复打开** | 已修复但验证不通过，重新打开 | 验收失败时由张良标记 |
| **修复中** | 研发正在修复 | 韩信/萧何认领修复时 |
| **已修复** | 修复完成，待测试验证 | 代码提交后标记 |
| **非 Bug** | 经验证不是 Bug，关闭 | 陈平确认误报后关闭 |
| **需求设计问题** | 问题根因在需求或设计 | 流转回需求侧，关联 Story |

#### 字段填写规范

| 字段名 | 类型 | 必填 | 填写规范 |
|-------|------|-----|---------|
| `Bug 标题` | Title | 是 | 【模块】简要描述 |
| `状态` | Select | 是 | 上述六种状态之一 |
| `严重级别` | Select | 是 | 致命/严重/一般/建议 |
| `发现人` | Person | 是 | 通常为陈平 |
| `责任人` | Person | 是 | 修复人 |
| `关联 Task` | Relation | 是 | 关联原始 Task |
| `关联 Story` | Relation | 否 | 关联需求 Story |
| `发现时间` | DateTime | 是 | 登记时间 |
| `修复截止时间` | DateTime | 否 | 根据严重级别自动计算 |
| `Bug 描述` | Text | 是 | 复现步骤、期望结果、实际结果 |
| `修复方案` | Text | 否 | 修复人填写简要方案 |
| `验证结果` | Select | 否 | 通过/不通过 |

#### 操作权限
- **曹参权限**：状态更新、责任人分配、严重级别调整、归档
- **陈平权限**：Bug 创建、验证结果更新、非 Bug 关闭
- **韩信/萧何权限**：修复中认领、已修复标记、修复方案填写
- **张良权限**：需求设计问题判定、重复打开标记

### 2.3 外部需求看板

#### 流程流转

```
外部消息 → 郦食其登记 → 夏侯婴读取/分发 → 曹参审核 → 进入迭代看板
```

#### 字段填写规范

| 字段名 | 类型 | 必填 | 填写人 | 填写规范 |
|-------|------|-----|-------|---------|
| `需求标题` | Title | 是 | 郦食其 | 外部需求简述 |
| `来源` | Select | 是 | 郦食其 | Slack/邮件/微信/其他 |
| `原始内容` | Text | 是 | 郦食其 | 原始需求文本粘贴 |
| `登记时间` | DateTime | 是 | 自动 | 创建时间 |
| `分发建议` | Select | 否 | 夏侯婴 | 技术类/产品类/运营类/待定 |
| `建议负责人` | Person | 否 | 夏侯婴 | 建议的角色 |
| `审核状态` | Select | 是 | 曹参 | 待审核/已采纳/已拒绝/需补充 |
| `审核意见` | Text | 否 | 曹参 | 审核结论和理由 |
| `关联迭代` | Relation | 否 | 曹参 | 纳入哪个迭代执行 |
| `优先级` | Select | 是 | 曹参 | P0/P1/P2/P3 |

#### 视图配置
- **待审核视图**：筛选 `审核状态 = 待审核`
- **已采纳视图**：筛选 `审核状态 = 已采纳`
- **来源统计视图**：按 `来源` 分组，统计外部需求渠道分布

## 四、与 OpenViking 记忆的联动方式

### 4.1 记忆同步机制

| 联动方向 | 触发时机 | 操作内容 |
|---------|---------|---------|
| **Notion → OpenViking** | Task 状态变更 | 将看板状态同步到记忆系统 |
| **OpenViking → Notion** | 记忆更新时 | 将关键决策同步到 Notion 备注 |
| **双向同步** | 11:00 定时检查 | 陆贾执行看板校验 |

### 4.2 记忆沉淀规则

| 看板内容 | 沉淀位置 | 沉淀时机 |
|---------|---------|---------|
| 迭代快照 | `memory/iterations/YYYY-MM-DD.md` | 每日 17:20 归档后 |
| 复盘记录 | `memory/retros/YYYY-MM-DD.md` | 每日 19:00 复盘后 |
| 流程改进 | `memory/process-improvements.md` | 提炼行动项后 |
| 度量数据 | `memory/metrics/weekly-YYYY-Www.md` | 每周五汇总后 |

### 4.3 关键记忆字段映射

```
Notion Task → OpenViking Memory
├── 任务名称 → memory.title
├── 状态 → memory.status
├── 负责人 → memory.assignee
├── 迭代日期 → memory.iteration_date
├── token 开销(实际) → memory.token_cost
├── 开始时间 → memory.start_time
├── 实际结束时间 → memory.end_time
├── 阻塞原因 → memory.blockers
└── 交付物链接 → memory.deliverable_url
```

### 4.4 记忆检索规范

- **晨会前**：从 OpenViking 检索昨日未完成 Task
- **日终评审前**：检索当日完成 Task 的交付物链接
- **复盘时**：检索当日所有 Task 的 token 开销数据
- **周报时**：检索本周所有迭代和复盘记忆
