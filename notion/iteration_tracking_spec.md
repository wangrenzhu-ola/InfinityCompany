## 1. Token 开销记录规范

### 1.1 Token 统计范围定义

| 指标 | 定义 | 说明 |
|------|------|------|
| 输入 Token | 发送给 AI 的 Prompt 文本 Token 数 | 包含系统提示词、上下文、用户输入 |
| 输出 Token | AI 返回的响应文本 Token 数 | 包含推理过程和最终答案 |
| 总 Token | 输入 + 输出 | 单次交互的完整消耗 |
| Task Token | 完成整个 Task 的累计 Token | 多次交互的累加值 |

### 1.2 Token 单位换算（Kimi API 示例）

```yaml
pricing:
  input_token_cost: 0.003  # 元/1K tokens
  output_token_cost: 0.006  # 元/1K tokens
  
cost_formula: |
  成本(元) = (输入Token × 0.003 + 输出Token × 0.006) / 1000
  
# 示例：一次交互消耗 输入10K + 输出5K
# 成本 = (10000 × 0.003 + 5000 × 0.006) / 1000 = 0.06元
```

### 1.3 Token 记录时机

```
Task创建时
    ↓ 预估Token数（基于复杂度）
每次交互后
    ↓ 记录本次交互Token数
阶段性检查点
    ↓ 每2小时检查累计Token数
Task完成时
    ↓ 最终Token总数统计
```

### 1.4 Token 预估 vs 实际对比

| Task复杂度 | 预估Token范围 | 允许偏差 | 超阈值处理 |
|-----------|--------------|----------|-----------|
| P3-简单 | 5K-20K | ±30% | 记录偏差原因 |
| P2-中等 | 20K-50K | ±25% | 记录偏差原因 |
| P1-复杂 | 50K-100K | ±20% | 需要复盘说明 |
| P0-紧急 | 100K+ | 需审批 | 必须事前审批 |

### 1.5 Token 超阈值告警规则

#### 单次交互告警
| 级别 | 阈值 | 处理动作 |
|------|------|----------|
| warning | 20K | 黄标提醒，关注上下文长度 |
| danger | 50K | 红标警告，建议拆分Task |

#### 单次 Task 告警
| 级别 | 阈值 | 处理动作 |
|------|------|----------|
| warning | 50K | 黄标，检查是否可优化 |
| danger | 100K | 红标，强制复盘分析 |
| block | 200K | 阻断，必须拆分Task |

#### 每日告警
| 级别 | 阈值 | 处理动作 |
|------|------|----------|
| warning | 200K | 当日Token使用偏高 |
| danger | 500K | 超出正常范围，需检查 |

### 2.2 工时计算公式（Notion Formula）

```javascript
// 计划时长(小时)
dateBetween(prop("计划结束时间"), prop("开始时间"), "hours")

// 实际总时长(小时)
dateBetween(prop("实际结束时间"), prop("开始时间"), "hours")

// 纯工作时长(小时)
dateBetween(prop("实际结束时间"), prop("开始时间"), "hours") - prop("中断时长/分钟") / 60

// 估时准确度
dateBetween(prop("计划结束时间"), prop("开始时间"), "hours") / 
(dateBetween(prop("实际结束时间"), prop("开始时间"), "hours") - prop("中断时长/分钟") / 60)
```

### 6.2 Notion Formula 公式集合

```javascript
// ==================== 基础时间计算 ====================

// 计划时长(小时)
prop("计划时长") / 3600000

// 实际总时长(小时)
(dateBetween(prop("实际结束时间"), prop("开始时间"), "hours"))

// 纯工作时长(小时)
(dateBetween(prop("实际结束时间"), prop("开始时间"), "hours")) - prop("中断时长/分钟") / 60

// ==================== 效率指标 ====================

// 估时准确度
round(prop("计划时长") / prop("纯工作时长") * 100) / 100

// Token使用率（%）
round(prop("实际Token") / prop("预估Token") * 100)

// Token效率
round(prop("产出代码行数") / prop("实际Token") * 1000 * 100) / 100

// ==================== 状态判定 ====================

// 进度状态
if(prop("状态") == "已完成", "✅ 完成",
  if(dateBetween(prop("计划结束时间"), now(), "hours") < 0, "🔴 延期",
    if(dateBetween(prop("计划结束时间"), now(), "hours") < 24, "🟡 即将到期",
      "🟢 正常")))

// Token告警级别
if(prop("实际Token") > 200000, "🔴 block",
  if(prop("实际Token") > 100000, "🟠 danger",
    if(prop("实际Token") > 50000, "🟡 warning", "🟢 normal")))

// 工时健康度
if(prop("纯工作时长") > 10, "🔴 过载",
  if(prop("纯工作时长") > 8, "🟡 注意", "🟢 健康"))

// ==================== 汇总指标 ====================

// 完成率（用于日报表）
round(prop("完成Task数") / prop("计划Task数") * 100)

// 准时率（用于日报表）
round(prop("准时完成数") / prop("完成Task数") * 100)

// Bug率（用于日报表）
round(prop("产生Bug数") / prop("完成Task数") * 100) / 100

// MTTR（小时）
if(prop("修复Bug数") > 0, round(prop("Bug修复总时长") / prop("修复Bug数") * 100) / 100, 0)
```
