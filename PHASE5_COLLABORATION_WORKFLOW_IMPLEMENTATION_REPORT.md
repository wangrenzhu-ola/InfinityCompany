| 章节 | 主要内容 |
|------|----------|
| **流程概述与目标** | 角色定位、响应时效目标（5分钟内响应）、流程边界 |
| **流程图** | Mermaid 格式主流程 + 看板流转子流程 |
| **详细步骤说明** | 5个核心步骤（接收→检索→判断→答复/登记→通知）的输入/处理/输出 |
| **决策节点说明** | 垃圾信息过滤规则（拦截率≥90%）、消息分类标准、匹配度计算规则（80%/50%阈值） |

| **外部需求分类标准** | 需求类型矩阵（产品功能/技术支持/商务合作/其他） |
| **数字分身答复模板** | 6套标准模板（欢迎语、功能咨询、需求已记录、投诉安抚、合作意向、无法答复） |
| **异常处理流程** | 6种异常场景、重试策略、4级降级模式 |
| **角色协作接口** | 与夏侯婴/陆贾/张良的协作流程图与接口定义 |
| **工具使用清单** | 核心工具、环境变量配置、API调用示例、监控指标 |

**关键流程链路**:
```
外部消息到达 → 渠道识别 → 垃圾过滤 → 消息分类 → 知识库检索 → 匹配度判断
    ↓ 能回答（≥80%）
数字分身直接答复
    ↓ 不能回答（<80%）
写入 Notion 外部需求看板 → 通知夏侯婴 → 等待 Owner 审核
```

## 🔄 完整协作流程链路

### 外部需求链路

```
外部用户/同事/合作方
        │
        ▼
    郦食其 (外部助理) ──┬── 直接数字分身答复（已知问题）
        │              └── 登记到 Notion 外部需求看板
        ▼
   Notion 外部需求看板
        │
        ▼
    夏侯婴 (私人助理) ── 09:15 定时读取 + 意图识别 + 分类整理
        │
        ▼
    你 / Owner (刘邦) ── 审核确认
        │
        ▼
   产品经理 / PMO / 技术负责人
```

### 内部交付链路

```
Owner 需求输入
    │
    ▼
夏侯婴 (私人助理) ── 意图识别 → 需求分类
    │
    ▼
曹参 (PMO) ── 建卡 → 创建 Story/迭代
    │
    ▼
张良 (产品经理) ── 产品定义 → PRD 输出
    │
    ▼
叔孙通 (设计师) ── 设计输出 → 设计稿
    │
    ▼
萧何 (架构师) ── 技术拆解 → Task 创建
    │
    ▼
韩信 (全栈研发) ── 研发实现 → 代码提交
    │
    ▼
陈平 (测试) ── 测试验证 → Bug 提交/修复
    │
    ▼
张良 (产品经理) ── 产品验收
    │
    ▼
周勃 (运维) ── 部署附着 → 上线
    │
    ▼
陆贾 (知识库管理员) ── 知识归档
```

## 📊 关键流程指标

### 外部助理流程

| 指标 | 目标值 | 说明 |
|------|-------|------|
| 响应时效 | ≤5分钟 | 外部咨询得到初步响应 |
| 需求登记率 | 100% | 不能即时答复的需求全部登记 |
| 垃圾信息拦截率 | ≥90% | 自动识别垃圾信息 |
| 流转效率 | ≤2小时 | 需求登记到 Owner 审核 |

### 私人助理流程

| 指标 | 目标值 | 说明 |
|------|-------|------|
| 意图识别准确率 | ≥95% | 正确理解老板指令 |
| 需求分发时效 | ≤5分钟（工作类） | 快速路由到对应角色 |
| 外部需求响应 | 每日09:15前 | 完成看板读取与分类 |
| 紧急上报时效 | ≤2分钟 | P0 事件直报老板 |
| 健康提醒执行率 | 100% | 定时触发健康关怀 |

### 交付闭环流程

| 指标 | 目标值 | 说明 |
|------|-------|------|
| 需求完成率 | ≥90% | 迭代内需求按时完成 |
| Bug 修复时效 | P0≤4h, P1≤24h | 不同严重程度响应时间 |
| 代码评审通过率 | ≥85% | 一次评审通过比例 |
| 测试覆盖率 | ≥80% | 自动化测试覆盖 |
